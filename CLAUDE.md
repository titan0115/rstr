# RSTR — local shader image tool (project guide)

Formerly **SHDR**. A local, brutalist, grayscale, shader-based image processor.
**Two heads, one shader codebase:** a double-click editor + a headless batch engine, sharing `src/` and one portable **"style code"**.

> Full version-by-version history & detailed spec: `docs/superpowers/specs/rstr-design.md`. **This file is the fast entry point** — read it first, dive into the spec for detail.

## Open source & deploy (public since 2026-07-15)

RSTR is a **public, open-source** project — **https://github.com/kirillpes/rstr**, licensed **GPL-3.0** (copyleft: any distributed fork/derivative must also stay open). The editor is deployed on **Vercel** at **https://rstr.vercel.app** (GitHub-connected — a push to `main` auto-deploys to prod, a PR gets an auto preview deploy).

**Owner workflow — commit & test exactly as before, zero PR friction.** Going open-source does NOT change how the owner works locally:
- Edit `index.html` / `src/*`, test by double-clicking `index.html` (or on the live site after a push).
- `git add -A && git commit -m "…"` → `git push` to `main` → Vercel auto-deploys. **Local == GitHub == live by construction** (the owner's stated invariant — keep it true).
- `main` is **deliberately NOT branch-protected** — the owner pushes straight to it, no self-PR. Do not propose enabling branch protection unless he asks.
- The ONE new habit, and only if outside contributors have merged PRs: **`git pull` before you push** (a push onto a stale `main` is rejected non-fast-forward). Nothing else about the loop changes.
- **Static deploy = editor only.** `.vercelignore` keeps `engine/`, `node_modules/`, `docs/`, `CLAUDE.md` out of the Vercel upload. The engine (`engine/rstr.js` — Node + puppeteer + system Chrome) stays a LOCAL batch head; it is not, and should not be, deployed to Vercel.
- Manual deploy fallback (if ever needed): `vercel --prod` from the project root. Outside contributors use **fork → PR**; grant direct push only by inviting a collaborator (`gh api -X PUT repos/kirillpes/rstr/collaborators/<user>`).

## Run it

- **Editor** — double-click `index.html`. No server, no build (classic `<script>`, works over `file://`), also hostable as static files. Desktop shortcut: `RSTR.lnk` (created 2026-07-10; target `index.html`, icon `icon/rstr.ico`).
- **Engine (batch)** — `cd engine && node rstr.js --preset <file|id> --in <dir> --out <dir> [--suffix -rstr]`. Headless **system Chrome** (puppeteer-core) running the SAME shaders. `--preset` = a `.json` path, a bare filename in `presets/`, or a bare `rstr_` id matched against `presets/*.json`. Output extension follows `output.format`.
- **From another Claude / any project** — the global skill **`rstr`** (`~/.claude/skills/rstr/`) batch-applies a style code to a folder. Say: `прогони папку стилем RSTR: {code}`.

## Architecture / key files

- `index.html` — editor entry; loads `src/*.js` (classic scripts → global `window.RSTR`).
- `src/gl.js` — WebGL2 helpers (compile, fullscreen triangle, ping-pong FBOs).
- `src/effects/` — **the EFFECTS registry** (classic scripts: `core.js` + category modules `tone`/`halftone`/`color`/`distort`/`stylize` + `index.js`): one entry per effect `{id,name,category,params[], frag OR cpu, presets[]}`. UI is generated from `params`. **Adding an effect = one entry.** Param types: `range` / `select` / `color` / `text` (string, cpu-only — never becomes a uniform; used by ascii's characterSet) / `stops` (array of `{pos,color}`; gradientmap's gradient stops — see `src/pipeline.js` below for how this becomes uniforms).
- `src/assets.js` — bundled image assets (tooooools' 6 pattern tiles + displacement map) as data URIs, decoded async into `RSTR.assets` (ImageData); `RSTR.assetsReady` promise — the engine gates `__RSTR_READY__` on it, the editor re-renders when it resolves. Data URIs keep `file://` untainted. (The displacement map is currently unused — `distort` was removed 2026-07-10; kept in case a distort-class effect returns.)
- `src/pipeline.js` — image→texture, runs the mix (ordered passes). Supports a per-effect **CPU stage** `cpu(rgba,w,h,params)` as an alternative to a `frag` shader (`frag`/`cpu` mutually exclusive; dither uses `cpu` for real Floyd–Steinberg). **Per-layer opacity + blend mode** ride one blend pass mixing a layer's output over its own input (GPU layers: scratch `blendFbo` then `BLEND_FRAG`, or `BLEND_MODE_FRAG` when `blend !== 'normal'`; CPU layers blended in JS against a pre-`cpu()` snapshot). A layer flagged **`mask`** feeds that same pass's opacity instead of compositing (`u_mask`/`u_hasMask`/`u_maskInvert`). The pinned **`◇ ORIGINAL`** row is a separate final pass — `PLATE_FRAG` composites the finished stack Porter-Duff source-over the raw source texture (`u_plate`) × opacity. `output.scale` mode **`width`** = an explicit working-buffer WIDTH in px. `computeGeometry` = OUTPUT crop/scale. `clearImage()`. A `type: 'stops'` param (currently only gradientmap's `stops`) uploads as three uniforms — `u_stopPos[8]`/`u_stopColor[8]`/`u_stopCount`, sorted ascending and truncated to 8 with a `console.warn` — instead of the single `u_<key>` every other param gets; NOT cpu-only despite being array-shaped (that was only ever true because gradientmap was CPU pre-2026-07-13).
- `src/preset.js` — (de)serialize style code, deterministic `rstr_` id (cyrb53), style-library + per-effect preset localStorage, `parseStyleCode`. Layer `opacity`/`blend`/`mask` and the top-level `source` block each serialize **only when non-default** (old codes/ids stay byte-identical); `BLEND_MODES` is the single source of truth for both the id list and the shader's numeric `u_blend` index. Unknown effects in a pasted code are dropped with a console warning, not a crash.
- `src/ui.js` — the entire editor UI. `makeScrub` = the shared scrub-bar control (click-to-type readout, Shift = 10× step, opt-in magnetic `snap`). Blend-mode picker is a custom hover-preview popover (`buildBlendDropdown`), not a native `<select>`. The `/Effects` list is user-organisable (drag-to-reorder + collapsible groups: `rstr.effectOrder`/`rstr.effectGroups`).
- `engine/rstr.js` + `engine/render.html` — batch head; `render.html` loads the SAME `src/*` and exposes `window.RSTR.render`.
- `presets/` — example style codes. `test-assets/` — sample images. `icon/` — app icon (`rstr.ico`, source `icon01.png`); root `favicon.ico`.

## Editor model (current state)

- **Three columns** (2026-07-12 rebuild — the old single 280px column overflowed and clipped its own footer): **nav** (`#nav-panel`: `RSTR` header → `/Styles` fixed header — library dropdown + Save/Del + Copy code/→ Presets + a collapsed `Backup` disclosure (Export all/Import) + Reset — → `/Effects`, the **only scrolling region**, styled square scrollbar) · **workbench** (`#panel`: PREPROCESS, always expanded → the EDIT/NEW/OUTPUT settings panel with its own RESET + a PRESETS modal trigger → `ADD TO MIX` → LAYERS/MIX stack, `◇ OUTPUT` pinned top → **pinned footer**, `EXPORT` only) · **canvas**. `⚙` (canvas top-right) still swaps `/Effects` for the enable/disable checklist, same order/groups. `PASTE CODE` (2026-07-13) was removed from `/Styles` — see the bullet below.
- **Canvas overlays**: `✕ New image` top-left · **`⚙` settings gear top-right** (toggles the effect enable/disable checklist in the nav column) · zoom controls bottom-left.
- **Scrub bars** (`makeScrub`) — THE range control everywhere (PRE + all effect `range` params; replaced rotary knobs and native sliders): full-width bar, label left / value right, fill = position; click-or-drag anywhere = absolute set, wheel = ±step, dblclick = reset to default. **Click-to-type** is the default on every value readout (opt out per-instance) — clamped + rounded to the param's own precision, never snapped, Esc cancels. **Shift** (drag or wheel) quantizes to 10× step. Opt-in **magnetic `snap`** (PRE's Canvas scrub uses it): travels through every value at full step resolution but sticks within a tolerance of 8 to a multiple of `snap` — not rounding, which made intermediate values unreachable. **2026-07-13 fix — top quarter of every range was unreachable by drag:** `.scrub-value` (the readout, occupying the rightmost ~25% of the bar) used to `stopPropagation()` on its own pointerdown, so a drag starting there never reached the bar. Fixed by arming a normal drag on ANY pointerdown (including on the readout) and, only when the press started on the readout, holding the first value commit back until the pointer moves past a 3px tolerance — a plain click still opens the type-in editor, real movement graduates straight into an ordinary drag.
- **Editable layers:** click a mix layer → `EDIT · <fx> [n]` (edit in place, live); click an effect in the list → `NEW · <fx>` (preview on top). **Auto-commit (2026-07-15):** the FIRST change to any param of a NEW effect auto-commits it as a layer and switches to `EDIT` of it in place (`commitNewOnEdit`/`afterParamEdit`) — committed **by reference** so an in-flight scrub drag keeps mutating the same object with NO `buildActiveParams` rebuild mid-drag (a rebuild would drop the pointer capture); only `buildMixList`/`buildEffectList`/`refreshTargetHeader` run. Just selecting an effect and not touching a param commits nothing (free browsing preserved). `ADD TO MIX` stays as the explicit alternative and does the same commit + `EDIT`-jump (staying in NEW would double-apply the preview on top of the layer just added).
- **Per-layer compositing controls live on the MIX row itself**, not the edit panel (opacity since 2026-07-11, blend/mask since 2026-07-12 — layer-level, not effect params). Each row is two lines: the effect row, then a controls row with **MASK** toggle, **blend-mode** picker, and an **Opacity** scrub (0–100%; `mix(input, blend(input, effect(input)), opacity)`). `◇ OUTPUT` gets none of this (not blendable); a NEW (uncommitted) layer gets none either — it lands in the mix at 100%/Normal on `ADD TO MIX`.
- **Blend modes** — Figma's catalog minus `PASS_THROUGH` (groups-only): Darken/Multiply/Plus darker/Color burn · Lighten/Screen/Plus lighter/Color dodge · Overlay/Soft light/Hard light · Difference/Exclusion · Hue/Saturation/Color/Luminosity, plus RSTR's own **`Alpha`**/**`Alpha invert`** (the layer's grayscale output becomes an alpha mask on its backdrop). The picker is a **custom popover**, not a native `<select>` (a native select fires no hover events on its options) — hovering a row live-previews it on canvas; leaving/Esc/outside-click reverts; click commits.
- **MASK** — flagging a row means it is NOT composited; its raw output's luminance instead multiplies the blend opacity of the NEXT enabled, non-mask layer (an `INV` toggle appears alongside — stipple/dither marks are black). The masked target shows a `MASKED` badge. A trailing mask with nothing after it is a no-op; two masks in a row → the last one wins. Only reaches the adjacent next layer (see Backlog).
- **LAYERS rows drag-to-reorder** (2026-07-13, same idiom as `/Effects`: native HTML5 DnD, 1px insertion line, no library) — the drag SOURCE is the WHOLE row (no grip handle; an earlier same-day version used a dedicated `.mix-drag-handle`, since removed). A capture-phase pointerdown guard (`wireMixRowDragToggle`) recomputes `row.draggable = !e.target.closest('.scrub, .icon-btn, .mix-blend')` (`MIX_ROW_INTERACTIVE_SELECTOR`) on every press — so a press on the opacity scrub, eye toggle, MASK/INV, or the blend-mode trigger flips the row briefly non-draggable and never starts a drag, while a press on the row's label/background drags normally. `moveLayerTo` tracks the currently-EDITed layer by object identity so a reorder never silently re-targets EDIT onto a different layer. The old ↑/↓ reorder buttons are GONE. `◇ OUTPUT`/`◇ ORIGINAL` stay pinned and non-draggable (a `.mix-drag-spacer` keeps their eye icon column aligned with draggable rows' column) but remain valid DROP targets at their boundary.
- **Per-layer enable is now an eye toggle** (`makeEyeToggle`, open eye = visible / slashed eye = hidden) — same underlying `layer.enabled` / `state.outputEnabled` / `state.source.enabled` booleans, no schema change, just a brutalist icon button replacing the native checkbox (no browser chrome, no accent color). `◇ OUTPUT` and `◇ ORIGINAL` use the same control.
- **`◇ ORIGINAL`** — pinned BOTTOM row in LAYERS, counterpart of `◇ OUTPUT` on top: a base plate, the finished stack composited Porter-Duff source-over `original × opacity`. Not part of `state.mix`, not draggable/removable, no blend control. **Defaults OFF** — see Decisions.
- **PREPROCESS module** (renamed from `PRE` 2026-07-15) (Canvas / Blur / Grain / Gamma scrubs + black/white-point levels track):
  - **Canvas** (renamed from Scale, 2026-07-12) writes `output.scale = {mode:'width', size}` directly — an explicit working-buffer WIDTH in px, not a percentage. On image load it defaults to the source's own native width (opens 1:1); its range is source-dependent (`100..2×sourceWidth`, rounded to the nearest 100) and magnetically snaps to multiples of 100. A loaded/pasted style code's own `output.scale` still wins. `none`/`fit`/`exact` modes are untouched — see Decisions for why `width` exists.
  - Blur/Grain/Gamma/BP/WP = the live `state.pre` working buffer edited through this module. **2026-07-15 model change (auto-commit):** the moment `state.pre` goes non-identity it **auto-commits a single `preprocess` layer at the FRONT of the stack** (applied first — global preprocessing), edited in place via this same module. `reconcilePreLayer` mirrors `state.pre` → the layer's params on every change and tracks it by reference as `preLayerRef`; reverting all scrubs to identity (or RESET) removes the layer; the front-insert/remove adjusts `editTarget.index` in both directions so EDIT never silently re-targets. **No fragmentation** (one layer, not one-per-tweak), no reset, and **no longer glued under the next `ADD`ed effect** — PREPROCESS is now an independent stack layer (`addToMix` no longer bakes it). `Canvas` is NOT part of `state.pre` (it writes `output.scale`) so it never triggers a commit. Exports as a normal `preprocess` stack layer. Legacy codes with a `pre` block still load (converted to a head `preprocess` layer; the editor sets `preLayerRef`/`state.pre` from it so the module edits it live; the engine prepends it — same look either way).
- **`◇ OUTPUT`** = pinned top layer: crop ratio + **3×3 Figma-style align** + scale (none / longest-side / **width** / exact W×H) + format/quality + live `SRC→OUT` dims. Toggle = passthrough.
- **Canvas viewport** (display-only, never affects export): fit-on-load, zoom slider + Fit/100%, **Ctrl+wheel** zoom-to-cursor, **middle-drag** pan. `refitView()`'s never-upscale cap only fires on a genuinely NEW image (`view.freshImage`, capped at 100% so a small source opens 1:1). A buffer resize from a CANVAS drag on the SAME image instead **snaps the view to 100% zoom** (2026-07-13, REPLACES the earlier zoom-compensation behavior) — the on-screen image shrinks/grows right along with the working buffer, so the effect's marks hold their true pixel size while the picture scales around them, showing the effect coarsen/refine relative to the content instead of just holding the on-screen size constant. Canvas has `image-rendering: pixelated`.
- **Global Ctrl+V** — pastes an image (loads as source, same path as drag-drop) or a style (`rstr_` id / JSON, same path as `PASTE CODE`) anywhere on the page; junk shows a toast; paste inside a real `<input>`/`<textarea>` (inline widgets, scrub click-to-type, color-picker hex field) is left alone. Reads the `paste` `ClipboardEvent`, NOT `navigator.clipboard.read()` — the latter is permission-blocked on `file://` and would break double-click-to-run.
- **Undo / redo (2026-07-15)** — `Ctrl/Cmd+Z` undo, `Ctrl/Cmd+Shift+Z` / `Ctrl/Cmd+Y` redo. Snapshots the DOCUMENT only (`state.mix`/`output`/`outputEnabled`/`pre`/`source` as a JSON string — NOT `editTarget`, the UI cursor). **Debounced + diff-based:** `scheduleHistorySync()` (hooked into `requestRender()`/`requestOutput()`, plus the quality `input` handler which uniquely never re-renders) fires a 300ms-debounced `commitHistoryIfChanged()` that only records when the snapshot differs from the baseline — so a continuous scrub DRAG coalesces into ONE undo step and redundant hook calls are free. `applyHistoryState` restores the five fields, re-derives `preLayerRef` from the head `preprocess` layer (same rule as `applyStyle`), clamps a now-out-of-range `editTarget` to OUTPUT, and rebuilds every surface (`buildEditor`/`buildEffectList`/`buildMixList`/`syncPreUI`/`requestOutput`). Stacks capped at 50. The keydown handler reuses `handleGlobalPaste`'s `isEditableTarget` guard, so `Ctrl+Z` inside any real text field does the browser's NATIVE undo, not a document jump. Baseline established once at the end of `boot()`.
- **`/Effects` catalog is user-organisable** — drag-to-reorder, user-created collapsible groups (groups themselves draggable, renameable by double-click, no nesting). Persisted as `rstr.effectOrder` + `rstr.effectGroups`; robust to registry churn (stale ids ignored, missing effects appended, nothing lost).
- **Pure-grayscale brutalism**: monospace, square corners, 1px borders, no color/shadow/gradient/transition. Tight 10–11px type scale is intentional (design hooks flag it; ignore).

## Effects (20)
`adjust, levels` · `halftone, dither, stipple, dots, patterns, gradients, crosshatch, ascii` · `posterize, gradientmap` · `pixelate, scatter, glitch` · `aberration, crt, bevel, cellular, bloom`.
- **`posterize`** (2026-07-13 merge) — absorbed the former standalone `threshold` and `quantize` behind a `mode` select (**Posterize** / **Threshold** / **Quantize**), same pattern as the `dots` merge. Why merge: `quantize` computed the exact same `floor(col*levels+0.5)/levels` as `posterize`, just with hash-noise dither applied first — a near-duplicate registry entry; `threshold` is a different luma `smoothstep` cut but the same "reduce tonal levels" family. All three shader bodies moved in VERBATIM behind `u_mode`; `levels` range is the union (2–32); `mode` defaults to 0 so a pre-merge `posterize` code (no `mode` key) renders byte-identical. Migrated via `LEGACY_EFFECTS` (see Conventions) — a mix with old `threshold`/`quantize` layers gets a NEW `rstr_` id.
- **`halftone`** now carries a `mode`: **Mono** (0, the original single rotated dot screen — cell-centre luminance → mark size/shape at one screen angle) / **Riso** (1, 2026-07-13, MERGED IN from the former standalone `riso` effect — see Decisions for why). `default: 0` on `mode` is load-bearing — every pre-existing `halftone` code (no `mode` key) renders byte-identical (SHA-256-verified). `cellSize` is SHARED between the two modes (both already read the same screen-pitch uniform pre-merge). Riso mode's mechanism: **misregistration** — a per-ink CONSTANT hash-derived offset applied to BOTH the halftone-grid sampling origin and the source-color sample coordinate, so a whole ink separation shifts as one rigid unit (a real misregistered print pass, not a blur); **overprint** — each ink composites as a translucent multiply over a `paper` base color, so overlapping screens produce a genuine third mixed colour. Riso's three screen angles (15°/75°/45°, the classic print angles chosen to avoid moiré) are internal constants, not a param; `angle` (Mono's own screen angle) is gated `showIf mode:[0]`. Params: `inkCount` (2/3), `ink1`/`ink2`/`ink3` (color), `paper` (color), `registration`, `inkTexture`, `seedOffset`. **`ink3` is gated on `mode:[1]`, not on `inkCount:[3]`** — `showIf` only supports ONE controller key (see Conventions), so it can't be conditioned on "mode is Riso AND inkCount is 3" at once; the coarser gate means Ink 3's swatch shows even when `inkCount` is 2 (just unused there). Deterministic hash, never `Math.random`. Colored output is intentional in Riso mode — like `gradientmap` (whose Hue/Saturation `map` modes carry recolor's old coloring behavior forward), the editor UI stays grayscale but the pixels needn't.
- **`gradientmap`** (2026-07-13 merge) — absorbed the former standalone `recolor` OUTRIGHT, not as a `mode` select like the dot/posterize/halftone merges: `gradientmap` was a CPU 256-entry LUT indexed by a variable-length `stops` array, `recolor` a GPU frag with 3 hardcoded stops plus `map` (Brightness/Hue/Saturation), `posterizeSteps`, noise (intensity/scale/gamma) and `repetitions` — `frag`/`cpu` are mutually exclusive in this registry, so one engine had to win outright; GPU did (see Decisions). `stops` is now a `type: 'stops'` param (≤8 `{pos,color}` entries, sorted ascending, uploaded as `u_stopPos[8]`/`u_stopColor[8]`/`u_stopCount` — see `src/pipeline.js`). `map` gained a 4th option, **Luminance** (value 3, the weighted `0.299/0.587/0.114` formula), now the DEFAULT so a bare pre-merge `gradientmap` code (no `map` key) still reproduces its old CPU LUT's own formula — a real bug was caught here mid-merge, see Decisions. `posterizeSteps`/noise/`repetitions` carry recolor's own identity defaults (255/0/0.3/1/1) so a gradientmap code that never touches them runs noise → posterize → repeat as no-ops. `recolor` itself is deleted; `LEGACY_EFFECTS` migrates old `recolor` codes via a `transform()` (stop1/pos1..stop3/pos3, pos 0–100, → a `stops` array, pos 0–1, plus an explicitly injected `map: 0` — see Conventions).
- **`invert` folded into `adjust`** (2026-07-13, same precedent as hue/sharpen's 2026-07-10 absorption) — `amount`/`channels` become adjust's `invert`/`invertChannels` (same 0–1 / 0–3 encodings). Sits **last in the chain, after gamma** (sharpen → hue → tone → invert): the old standalone `invert` was always its own trailing layer, so inverting the fully tone-adjusted result is the closest two-layer equivalent. Default `invert: 0` = identity, so every pre-existing `adjust` code (no invert keys) renders byte-identical. `invert` itself is deleted; `LEGACY_EFFECTS` migrates old `invert` codes via a `transform()` (see Conventions).
- **`glitch`** (2026-07-12) — horizontal slice displacement (VHS-tear / datamosh). Params `amount` (% chance a given slice tears), `sliceHeight`, `maxShift` px, `seedOffset`, `blockNoise` (subdivides a torn slice into narrower, independently-jittered blocks; 0 = one clean bar), and (2026-07-13) `chromaShift`. Torn rows sample `v_uv.x` shifted and **wrapped** (`mod`), not clamped or voided — distinct from CRT's alpha-void lens choice. Deterministic via a coordinate hash, never `Math.random`. `seedOffset` is a per-layer knob kept deliberately separate from the shared `u_seed` uniform — a param literally named `seed` would resolve to the same uniform location and silently shadow it.
  - **`chromaShift`** (2026-07-13) — a per-slice chroma split, PERCENTAGE OF THE SLICE'S OWN DISPLACEMENT (not an absolute px offset): R samples at `+shiftPx·chromaShift/100`, B at `−shiftPx·chromaShift/100`, same +R/−B convention as `aberration`. That's the whole point — stacking `glitch → aberration` gives a uniform split across the WHOLE image regardless of which rows tore, whereas real glitch splits color only INSIDE the torn slices, proportional to each tear; untorn rows stay bit-identical at any `chromaShift` (verified). Per-block variation falls out naturally when `blockNoise` is on. Default 0 = identity, so pre-2026-07-13 glitch codes render byte-identically. **Supersedes the earlier "deliberately does NOT split RGB" note** — that reasoning (aberration already owns it, just stack the two) was wrong: a whole-frame effect can't reproduce a per-slice-proportional split at any settings.
- **2026-07-10 consolidation:** removed `displace`, `distort`, `blur`, `hue`, `sharpen` as standalone effects — **hue + sharpen live inside `adjust`** now (order: sharpen → hue → tone; old adjust codes render identically, new keys default to identity); blur lives in PRE. Old style codes referencing removed effects: those layers are dropped by `validatePreset` with a console warning.
- **`dots`** — the whole dot family in one effect, `mode` = **Dots** / **Edge** / **Pattern** (2026-07-11 merge of the old `dots`, `edge`, `dotpattern`; see Decisions). One `frag`, three verbatim shader bodies behind `u_mode`. `gridType` is now Regular / **Staggered** (the old Benday offsets alternate rows in X, Brick offsets alternate columns in Y — same idea, same value `1`, so no remap). Params are gated per mode by **`showIf`** (see Conventions). Pattern mode = the old square-dot grid sibling of stipple (cell-center luminance → dot size on both axes; `spacing` = grid pitch px, `scale` = dot side at black as % of cell).
- **`aberration`** (2026-07-10) — chromatic aberration extracted from CRT (its 0.1-strength convergence was near-invisible): **Radial (lens)** mode = R/B shift grows from center to `amount` px at edges; **Linear** = uniform shift along `angle`. R +offset, B −offset, G fixed.
- **tooooools.app ports** (2026-07, from their JS bundles): `dots` (all three modes), `patterns, gradients, bevel, cellular` exact, `recolor` too (its math lives on inside `gradientmap` since the 2026-07-13 merge); `crt, ascii` rewritten; `dither, stipple` param-aligned. Originals' quirks kept on purpose (dead Threshold in color Bayer/Random dither, gradients' NaN bottom band, non-serpentine F-S, bevel's untouched 1px border).
- **crt** — tooooools port **with four deliberate departures**: R/B convergence extracted to `aberration`; `distortion` (now 0–2) replaced with a normalized tube curvature — multiplier 1 at edge midpoints, center magnifies (visible bulge), corners sample outside the source → **transparent alpha void, not edge-stretched pixels**; `maskStrength` (2026-07-11) dials the phosphor mask down to nothing (`pattern = mix(vec3(1.0), pattern, u_maskStrength)`, default 1 = the port's look); and **`Scanlines`** (2026-07-12, value 3) — plain horizontal bands, no phosphor triads, reusing `dotPitch`/`dotScale` instead of new params. Mask picker is now Monitor(0) / LCD(2) / Scanlines(3) — **`TV`(1) dropped from the picker** (its only difference from LCD was a half-element vertical phase shift of alternate subpixel-column groups; measured mean channel diff 4.5/255, no better at coarse pitch) but the shader branch and value stay so old codes carrying `patternType:1` still render byte-identically — hence the deliberate gap. Glow, bloom (Screen/Light/HDR); brightnessBoost 2.5 hardcoded like theirs.
- **dither** — pattern F-S / Bayer / Random + pixelSize / colorMode / colorCount / threshold. F-S = real CPU-stage error diffusion; Random uses a coordinate hash, not Math.random (preview == batch).
- **stipple** — threshold / gridType Regular·Benday / gridAngle ±90 (widened from ±45 on 2026-07-15) / x·ySquares / min·maxWidth px. Only WIDTH is brightness-mapped → variable-width bars (like the original). **2026-07-12 seam fix:** dropped the Y-axis antialias falloff (`mask = mx`, was `mx * my`) — each fragment belongs to exactly one cell, so an AA'd Y edge faded to 0.5 exactly at the cell border with no neighbour to blend against, leaving a grey seam every `cellY` px; the bar now fills the full cell height by construction.
- **ascii** — CPU canvas-text: columns / rows / characterSet (free-text param) / comments / borders. Black on white.
- **cellular** — CA types Classic / LTL / MNCAB / MNCC; exact rules via summed-area tables (verified bit-for-bit).
- **patterns** reads `RSTR.assets` (see `src/assets.js`). **`patterns`/`gradients`/`scatter` (2026-07-12) preserve incoming alpha** — all three are CPU stages that draw onto an opaque white canvas and used to force `a=255` on their output, silently punching out the new `◇ ORIGINAL` plate and any `alpha`/`alpha-invert` holes downstream; they now restore the source pixel's own alpha after drawing. RGB output is unchanged (byte-identical on opaque sources).

## Conventions & gotchas

- No build, no framework, no ES modules (classic scripts so `file://` double-click works). Global `window.RSTR`.
- localStorage keys are `rstr.*` (`rstr.styleLibrary`, `rstr.disabledEffects`, per-effect presets). Style id prefix `rstr_`.
- Engine hardcodes the system-Chrome path + SwiftShader/ANGLE GL flags.
- **Style Library lives in the browser's localStorage.** `/Styles` bridges it to disk: **`→ Presets`** writes the selected style as an engine-ready `presets/<name>.json` (File System Access API — `file://` IS a secure context and `showDirectoryPicker` works there; the chosen dir handle is cached in IndexedDB so later exports need no dialog, with an `<a download>` fallback if the picker is unavailable/denied). **`Export all` / `Import`** back up and restore the whole library as one JSON — localStorage is the ONLY copy otherwise, so a cache clear would wipe it. Both live behind a collapsed **`Backup`** disclosure toggle (2026-07-13, `toggleBackupRow`) since they're needed once in a blue moon, not every session. **`PASTE CODE`** (a dedicated button doing the same thing) was removed the same day — the global Ctrl+V handler already runs clipboard text through the identical `parseStyleCode`/`applyStyle` path from anywhere on the page, so the button was redundant. Conceptually: a style code IS the JSON — `Copy code` and `→ Presets` are the same data going to two destinations (clipboard vs. disk), not two formats; the `rstr_` id, by contrast, is a REFERENCE (a hash) that only resolves if that JSON exists somewhere (a `presets/*.json` for the engine, a library entry for the editor).
- `--preset` accepts a `.json` path, a bare name in `presets/` **with or without the `.json` extension**, or a bare `rstr_` id.
- OUTPUT processing order: `input → crop → scale → effects → encode` (so preview == batch).
- A disabled effect (⚙) is only hidden from the picker; it still renders inside a loaded mix.
- Layer schema: `{effect, enabled, params, opacity?, blend?, mask?}` — `opacity` only when < 1, `blend` only when ≠ `normal`, `mask` only when present (an object `{invert}`). The top-level `source` block (the `◇ ORIGINAL` plate, `{enabled, opacity}`) serializes only when ≠ its `{enabled:false, opacity:1}` default. All four rules exist for the same reason: every pre-existing style code's `rstr_` id stays byte-identical (re-verified after every change in the 2026-07-12 session). `preprocess` is a normal (internal) registry entry so it can sit in a stack.
- **`showIf` on a param is UI-ONLY** (`showIf: { key: 'mode', in: [0, 1] }` → the edit panel skips the control when the gate fails, and rebuilds when the gating select changes). The pipeline still uploads EVERY param as a uniform regardless of mode — do not "optimize" that away or GL gets unset uniforms.
- **Legacy effect renames go in `LEGACY_EFFECTS` in `src/preset.js`** (`edge → dots{mode:1}`, `dotpattern → dots{mode:2}`, `threshold → posterize{mode:1}`, `quantize → posterize{mode:2}`, `riso → halftone{mode:1}`, and, 2026-07-13 later the same day, `recolor → gradientmap` and `invert → adjust`), applied in `validatePreset` *before* the unknown-effect drop. Each entry is either `{ effect, params }` (old param keys carry across unchanged, only a `mode` key is new — every mode-select merge above) or, since the recolor/invert merges, `{ effect, transform(params) }` for a merge whose params need actual reshaping, not just a key union: `recolor`'s fixed `stop1/pos1..stop3/pos3` (pos 0–100) become `gradientmap`'s `stops` array (pos 0–1) plus an explicitly injected `map: 0`; `invert`'s `amount`/`channels` become `adjust`'s `invert`/`invertChannels`. A legacy entry has exactly one of `params` or `transform`, never both. A migrated mix gets a NEW `rstr_` id (the hash covers the mix) — that's expected, and applies the same way to every merge in this list, not just the first one.
- **`type: 'stops'` is no longer cpu-only** (2026-07-13) — that was only ever true because `gradientmap`'s CPU LUT was its sole user; it's now uploaded as GPU uniforms too (`u_stopPos[8]`/`u_stopColor[8]`/`u_stopCount`, see `src/pipeline.js`). A `stops` array longer than 8 is sorted ascending then truncated with a `console.warn`, never a crash.
- **`showIf` supports exactly ONE controller key** (`{ key, in: [...] }`, no AND-composition) — a real mechanism limitation, not a design choice: `halftone`'s `ink3` wants to show only when Riso mode AND `inkCount:3`, but can only gate on one of them (chose `mode`, the coarser condition). Extending `showIf` to multiple keys would fix this; not done, so a handful of params show slightly more often than strictly necessary.
- Param-set churn: the 2026-07 tooooools port changed `crt`/`dither`/`stipple`/`ascii` params; 2026-07-10 removed crt's convergence keys and changed the meaning of its `distortion`; 2026-07-11 merged the dot family and added crt's `maskStrength`; 2026-07-12 added crt's `Scanlines` mask + dropped `TV` from the picker (value kept) and fixed stipple's Y-axis seam; 2026-07-13 merged `threshold`/`quantize` into `posterize`, added `glitch`'s `chromaShift`, then later the same day merged the short-lived standalone `riso` effect into `halftone` as a mode and deleted the short-lived `warp` effect outright (see catalog/Decisions). Old codes never crash — stale keys are ignored, missing keys default.
- **`seedOffset` (glitch) is deliberately not named `seed`** — `u_seed` is a shared uniform already uploaded by the pipeline for every pass; a param keyed `seed` would resolve to that same uniform location and silently shadow it.
- **crt's `patternType` has a deliberate value gap**: 0 Monitor, 2 LCD, 3 Scanlines — 1 (`TV`) is removed from the UI picker but the shader branch and value remain so old style codes carrying `patternType:1` still render.

## Decisions (what & why)

- **Name = RSTR** (vowel-dropped "raster"): brutal, a pro/legible term, reel-friendly. Chosen over *Raster* (future vector/SVG collision), *FRAG*, *SHADERWERK*.
- **Two heads / one shader codebase** (DRY): editor + engine both run `src/`, so batch output == on-screen preview by construction.
- **Classic scripts, not ES modules:** enables double-click `file://` + hosting with no build (Chrome CORS-blocks module imports over `file://`).
- **Style code = portable data recipe** (`rstr_` hash of the mix + output) → handed to another Claude via the `rstr` skill.
- **dither = real Floyd–Steinberg via a CPU stage** (sequential error diffusion can't be single-pass GPU). Bayer/Random stay in-shader.
- **OUTPUT is a layer**, with crop/scale **before** effects for WYSIWYG; Figma-style 3×3 crop anchor.
- **Scrub bars over knobs/sliders (2026-07-10):** rotary knobs dragged vertically were awkward and native range inputs tiny; one shared control with a full-width hit area, wheel nudge and dblclick-reset replaced both.
- **PRE bakes into the mix (2026-07-10; superseded 2026-07-15):** originally PRE was a working buffer for the *next* layer — on ADD it committed as a `preprocess` layer glued under that effect and reset. **2026-07-15 replaced this with auto-commit:** PRE (renamed **PREPROCESS**) is now an INDEPENDENT `preprocess` layer at the FRONT of the stack (applied first, global), auto-committed the instant it's used and edited live in place via the PREPROCESS module — no gluing, no reset, no fragmentation (`reconcilePreLayer`). Still keeps preview == committed result and makes PRE a normal visible/reorderable layer.
- **Engagement = commit (2026-07-15):** touching a control commits — the FIRST param edit of a NEW effect (and any use of PREPROCESS) drops it into the stack automatically, so `ADD TO MIX` is no longer the only path in. Chosen because requiring an explicit `ADD` after you've already dialed in a look was redundant friction; the by-reference commit (`commitNewOnEdit`) makes it survive an in-flight scrub drag with no panel rebuild. `ADD TO MIX` kept as an explicit alternative (commit without touching a control).
- **Opacity = pipeline blend pass, not per-shader:** one mechanism for all 25 effects (GPU scratch-FBO mix / CPU snapshot mix) instead of touching every shader.
- **Effect consolidation (2026-07-10):** hue/sharpen merged into adjust, blur/displace/distort deleted — fewer, denser tools over a long tail of one-slider effects.
- **Dot family → one `dots` with modes (2026-07-11):** `edge` was a strict param-*subset* of `dots` and `dotpattern` only swapped dot-size params for spacing/scale — three catalog rows for one idea. Merged behind a `mode` select (25 → 23 effects). The three shader bodies were moved **verbatim** into one `frag` behind `u_mode`, so output is unchanged — proven by rendering the old and merged shaders against the same params: **byte-identical** for all three legacy paths. Old codes migrate instead of breaking (`LEGACY_EFFECTS`); the price is a new `rstr_` id, accepted. Made `showIf` necessary: a 10-param union where any one mode uses 5–6 would have been a *worse* panel than three effects, so param visibility is now mode-gated.
- **CRT diverges from the port where the port was weak:** convergence → `aberration` (visible, reusable over any mix); distortion formula → normalized tube curvature (the original only cropped corners at usable strengths). Alpha void outside the tube instead of smeared edge pixels.
- **`output.scale` mode `width` exists (2026-07-12) because every ported effect param is in absolute pixels**, tuned against tooooools' ~600px canvas. RSTR's old default rendered at full source resolution, so on a 2400px photo a stipple cell reads 26px while the mark stays 1–4px — hairlines, the effect stops reading. `width` gives an explicit working-buffer WIDTH; PRE's Canvas scrub writes it directly, defaulting to the source's own native width so nothing changes until the user drags it. `none`/`fit`/`exact` stay as they were.
- **`◇ ORIGINAL` is a base plate, not a pipeline-input fade, and defaults OFF.** First built with "pipeline input" semantics — opacity fading the *source's* alpha before the stack ran — and that was wrong: it doesn't fill the holes `alpha`/`alpha-invert` punch downstream, it just fades the input. Replaced with a final Porter-Duff source-over composite of the finished stack on top of the raw source × opacity. Defaults off because crt already renders a deliberate transparent void outside its tube — a plate ON by default would silently fill that void and change every existing crt style code.
- **MASK is one bit, not Photoshop's masking machinery (2026-07-12):** the mix is a linear pipeline, so "reveal layer B through layer A" was inexpressible. A flagged layer is skipped by compositing; its luminance instead multiplies the blend opacity of the NEXT non-mask layer — folded into the existing blend pass, both GPU and CPU paths honour it. No editable mask channels, no linking, no group masks, and the mask only reaches the ADJACENT next layer (a distant target needs inter-layer pointers — backlog). Buys `stipple[MASK,inv] → gradientmap` for one bit of complexity instead of a real graph.
- **Blend-mode picker is a custom popover, not a native `<select>` (2026-07-12):** a native select's option list is browser-owned and fires no hover events on its options, so it can't drive the live-preview-on-hover the user wanted; a brutalist row-list popover can (`buildBlendDropdown`, a singleton re-targeted per open).
- **Blend rides the existing per-layer blend pass, not a shader touch:** same reasoning as opacity in v2.0 — one mechanism (`BLEND_MODE_FRAG`, W3C Compositing and Blending Level 1 ported verbatim, verified against an independent NumPy reference to ≤1/255) covers all 24 effects instead of 24 shader edits.
- **`threshold`/`quantize` merged into `posterize` (2026-07-13):** `quantize` computed the identical `floor(col*levels+0.5)/levels` as `posterize` plus one pre-step (hash-noise dither) — a near-duplicate catalog row, not a different idea. `threshold`'s math differs (a luma `smoothstep` cut) but it belongs to the same "reduce tonal levels" family, so folding it in via `mode` beats a fourth near-identical entry.
- **Merge by shared MECHANISM, not shared THEME** — the rule that makes `dots`/`posterize` good merges and would make a hypothetical `print` mega-effect (riso + ink bleed + paper scan) a bad one: ink bleed is diffusion/blur, paper scan is texture overlay, genuinely different computations wearing one `mode` select as a trenchcoat would be harder to reason about than separate small effects. **The rule was written and then immediately violated in the same session** — `riso` shipped as its own standalone catalog entry right after, on the (wrong) call that its print-specific mechanism didn't overlap anything else. The user caught it: `riso`'s actual mechanism — a rotated dot screen, cell-centre luminance → mark size — is the IDENTICAL mechanism `halftone` already used, just run 2–3 times at different fixed angles with a per-ink offset and composited as overprint. Same mechanism, not just same theme, so it was folded into `halftone` as a `mode` hours later (see the `halftone`/Riso entry above and the spec's `2026-07-13 revision` section). Lesson kept: apply the rule to the thing you just shipped, not only to the next proposal.
- **`gradientmap` absorbed `recolor` outright, not as a `mode` select (2026-07-13):** both were luminance/color-to-color remaps — same idea as the `dots`/`posterize`/`halftone` merges — but built on two INCOMPATIBLE engines (`gradientmap`: CPU 256-entry LUT; `recolor`: GPU frag), and the registry makes `frag`/`cpu` mutually exclusive. A mode-select merge (one effect, one engine, `u_mode` branching) was structurally impossible; one engine had to win outright. GPU won: recolor's shader already did real per-pixel gradient sampling with room for its extras (posterize/noise/repetitions); the CPU LUT could not absorb those without becoming a second per-pixel CPU loop duplicating work the GPU already does for free. The lesson: "merge by shared mechanism" (see the `riso`/`halftone` decision above) has a precondition — the mechanisms also have to share an ENGINE, or the merge is a replacement, not a unification.
- **First knowingly-broken byte-identity, and why it was accepted (2026-07-13):** the merged `gradientmap` initially defaulted `map` to recolor's own 0 (Brightness — unweighted `(r+g+b)/3`), which silently re-toned every pre-existing `gradientmap` code (it always indexed its LUT by WEIGHTED luminance, `0.299/0.587/0.114`) — max channel diff 113/255, mean 16, caught before it shipped. Fix: made `map: 3` (Luminance, the weighted formula) the new DEFAULT, so a bare gradientmap code (no `map` key) falls through to the formula it always had; the `recolor → gradientmap` migration now explicitly injects `map: 0` so migrated recolor codes aren't affected by the default moving out from under them. After the fix, old `gradientmap` codes are still NOT fully byte-identical — 3/255 max, mean 0.07–0.39 — because `posterizeSteps` (recolor's own default, 255, dividing by 254 not 255 — a deliberate faithful port of tooooools, load-bearing for recolor's OWN byte-identity) now runs for every `map` mode through the shared shader, where the old CPU LUT never saw a posterize step at all. Accepted as invisible (3/255 at mean 0.1) rather than removed, since removing it would regress `recolor`. This is the FIRST merge in this project to knowingly break byte-identity — every other migration (`edge`, `dotpattern`, `threshold`, `quantize`, `riso`, `invert`) stays exact.
- **Per-effect edge policy (void / clamp / wrap) is a deliberate call, never a global default** — first written as a `warp`-specific decision (`warp` was deleted 2026-07-13 on user feedback — he didn't want the geometric-distortion family; not being rebuilt), but the general precedent it recorded outlives the effect and still governs the two effects that ship it: `crt`'s tube-curvature distortion renders a transparent alpha void outside its bounded lens-disc (an optical remap of a disc has no data past its edge — clamping or wrapping would misrepresent that), while `glitch`'s slice tear instead wraps (`mod`) because a signal tear is a genuinely repeating artifact, not a bounded lens. Any future geometric-remap effect should make this call explicitly per mode, the way `crt` and `glitch` already do, rather than defaulting to one edge behavior for everything.
- **Pure-grayscale brutalism**; removed the orange accent; dropped effect categories (flat list).
- **Icon concept:** a living color image returned to a **1-bit dither** (Apple Lisa 1984 / Kare–Atkinson era); **dogfooded** (RSTR dithers its own icon). Final icon = user-made `icon/icon01.png`.
- **Project folder** renamed `ASCII → D:\01_PROJECTS\RSTR`.

## Backlog / deferred

Shipped work lives in the spec (`# v2.1 — DONE`, `# 2026-07-12 — DONE`, `# 2026-07-13 — DONE`, `# 2026-07-13 revision — DONE`), not here. This list is what's still OPEN.

- **Vector / SVG output** + "SCI" / generative effects (the RSTR name was picked to leave room for these).
- **Animation** (slide / stack, like tooooools) — needs a time parameter in the pipeline and a different export path (GIF/video). The biggest single piece left.
- **Inter-layer mask pointers** — MASK currently only reaches the ADJACENT next layer; masking a distant layer would need inter-layer pointers and a real graph, not the one-bit flag.
- Favicon crispness at 16px; per-effect polish.
- `assets.js` still bundles the (now unused) displacement map — restated honestly after `warp` was deleted (2026-07-13): there is currently NO effect that consumes it and no assumed replacement mechanism. The map stays an inert bundled asset until some future distort-class effect is proposed on its own merits and actually wants it — drop the asset if that never happens.
- **`warp` (the geometric-distortion family — Swirl/Pinch-Bulge/Ripple/Reeded Glass/Polar↔Rectangular) is explicitly NOT wanted** — the user rejected it and it was deleted 2026-07-13, same day it shipped. Do not rebuild it as part of the effect.app port backlog below or for any other reason without new explicit user ask.
- **effect.app port backlog** (2026-07-13, user wants these ported gradually): blur family, NTSC/dot-crawl, ink bleed, paper scan, curves/color-matrix/vignette folded into `adjust`. Geometric distortion is excluded — see above.
